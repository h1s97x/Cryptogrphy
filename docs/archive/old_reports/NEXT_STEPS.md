## 关于"需要这么多widgets吗？"

### 当前架构

每个加密算法都是一个独立的Widget：
```
Caesar_ui.py  → CaesarWidget
Vigenere_ui.py → VigenereWidget
AES_ui.py → AESWidget
...
```

### 是否可以简化？

**可以考虑的方案**：

1. **通用组件工厂**
   ```python
   # 不需要36个文件，只需要配置
   caesar_config = {
       "name": "Caesar Cipher",
       "algorithm": Caesar,
       "groups": [...]
   }
   widget = GenericCipherWidget(caesar_config)
   ```

2. **插件化架构**
   ```python
   # 每个算法是一个插件
   class CaesarPlugin(CipherPlugin):
       def get_config(self):
           return {...}
   ```

3. **保持现状**
   - 每个算法独立文件
   - 易于维护和扩展
   - 符合单一职责原则

**我的建议**：
- 短期：继续当前架构，完成组件更新
- 长期：考虑重构为插件化架构（阶段2-3）
