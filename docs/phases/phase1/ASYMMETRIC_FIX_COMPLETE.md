## 修复的问题

### 1. ✅ RSA加密模块修复

**问题描述**：
- RSA.py依赖缺失的mm_rsa模块
- 无法生成密钥、加密、解密

**解决方案**：
- 使用pycryptodome库替代mm_rsa
- 实现完整的RSA功能

**修复内容**：

#### 密钥生成
```python
from Crypto.PublicKey import RSA as CryptoRSA

key = CryptoRSA.generate(1024)
public_key = key.publickey()
keys = (public_key, key)  # (公钥, 私钥)
```

#### 加密功能
```python
from Crypto.Cipher import PKCS1_OAEP

cipher = PKCS1_OAEP.new(public_key)
encrypted = cipher.encrypt(data)
```

#### 解密功能
```python
cipher = PKCS1_OAEP.new(private_key)
decrypted = cipher.decrypt(encrypted_data)
```

**技术亮点**：
- 支持分块加密/解密（处理RSA 1024位密钥的长度限制）
- 最大明文长度：86字节
- 密文块大小：128字节
- 完整的错误处理和日志记录

---

### 2. ✅ RSA签名模块修复

**问题描述**：
- RSA_Sign.py依赖缺失的mm_rsa模块
- 无法进行RSA签名

**解决方案**：
- 使用pycryptodome的签名功能
- 实现RSA-PSS签名

**修复内容**：

#### 签名功能
```python
from Crypto.Signature import pkcs1_15
from Crypto.Hash import SHA256

# 计算SHA-256哈希
h = SHA256.new(data)

# 使用私钥签名
signature = pkcs1_15.new(private_key).sign(h)
```

**技术亮点**：
- 使用PKCS#1 v1.5签名方案
- 自动计算SHA-256哈希
- 签名长度：128字节（1024位密钥）

---

### 3. ✅ ECDSA UI模块修复

**问题描述**：
- ECDSA_ui.py导入了不存在的Hash.SHA256模块
- 导致ECDSA组件无法启动

**解决方案**：
- 修改导入路径使用core.algorithms.hash.SHA256
- 保持功能不变

**修复内容**：

```python
# 修复前
import Hash.SHA256.SHA256 as SHA2

# 修复后
from core.algorithms.hash import SHA256
```

## 技术细节

### RSA加密实现

**密钥长度**：1024位

**加密限制**：
- 最大明文长度：86字节（PKCS#1 OAEP填充）
- 密文长度：128字节

**分块处理**：
```python
max_length = 86  # 1024位密钥的最大明文长度
for i in range(0, len(data), max_length):
    block = data[i:i+max_length]
    encrypted_block = cipher.encrypt(block)
    encrypted_blocks.append(encrypted_block)
```

### RSA签名实现

**签名方案**：PKCS#1 v1.5

**哈希算法**：SHA-256

**签名长度**：128字节（1024位密钥）

**签名流程**：
1. 计算消息的SHA-256哈希
2. 使用私钥对哈希值签名
3. 返回签名（128字节）
