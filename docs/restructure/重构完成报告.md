## 发现的遗留问题

### 1. 非对称加密模块依赖缺失 ❌
**影响文件：**
- `core/algorithms/asymmetric/RSA.py`
- `core/algorithms/asymmetric/RSA_Sign.py`
- `core/algorithms/asymmetric/SM2.py`
- `CryptographicProtocol/Digital_Envelope.py`
- `CryptographicProtocol/Digital_Certificate.py`

**问题：**
这些文件依赖已删除的 `PublicKeyCryptography.mm_rsa` 模块，该模块在旧结构中提供RSA实现。

**错误信息：**
```
No module named 'PublicKeyCryptography'
```

**根本原因：**
- `mm_rsa` 模块在旧结构删除时一并被移除
- 新结构中的 RSA.py 只是一个包装器（wrapper），不包含实际实现
- 缺少底层RSA算法实现代码

### 2. ECC模块依赖缺失 ❌
**影响文件：**
- `core/algorithms/asymmetric/ECC.py`

**问题：**
依赖 `Crypto` 库（pycryptodome），但未安装或导入路径错误。

**错误信息：**
```
No module named 'Crypto'
```

### 3. 维吉尼亚密码方法签名不匹配 ❌
**影响文件：**
- `core/algorithms/classical/Vigenere.py`

**问题：**
`Thread.encrypt()` 方法签名与调用不匹配。

**错误信息：**
```
Thread.encrypt() takes 1 positional argument but 3 were given
```

### 4. UI组件导入问题 ❌
**影响文件：**
- `ui/widgets/__init__.py`

**问题：**
UI组件的 `__init__.py` 未正确导出 `classical` 等子模块。

**错误信息：**
```
cannot import name 'classical' from 'ui.widgets'
```
